# MyDemo Bot 项目文档

> 更新日期：2026-07-22

---

## 一、项目概述

基于 Spring Boot + MyBatis + LangChain4j，集成 iLink SDK 和 DashScope SDK，实现功能完善的微信智能助手 Bot。用户通过微信聊天即可与 AI 对话、查询天气、识别图片、生成图片、收听语音播报等。**全部用户操作均由 LLM Function Calling + @Tool 注解驱动，无硬编码关键词匹配。**

## 四、核心架构

### 4.1 请求数据流

```
微信消息 → WeChatBotService.handleMessage()
              │
      ┌───────┴────────┐
      ▼                ▼
  图片/语音/文件      纯文字 → CommandRouter.route()
  预处理后进route          │
                     ┌────┴────┐
                     ▼         ▼
               直接命令匹配   LlmService.chat() + 工具清单
                    │            │
                    │       LLM 返回 function_call
                    │            │
                    │    ToolRegistry.execute()
                    │     反射调用 @Tool 方法
                    │            │
                    └─────┬──────┘
                          │
                continueWithToolResult()
                 工具结果以 role:tool 回填 LLM
                        (ReAct循环)
                          │
                   LLM 生成自然语言回复
                          │
                          ▼
                   sendReply()
                   检查 ToolExecutionContext
                   → 图片生成/文件导出/重识别/发送文字
```

### 4.2 ReAct 循环

```
第1轮: LlmService.chat() + 工具清单
         → LLM 判断需要工具 → 返回 function_call
         → ToolRegistry 反射调用 @Tool 方法
         → 工具返回结果

第2轮: continueWithToolResult()
         → 将工具结果以 role:tool 回填 LLM
         → LLM 看到工具结果 → 生成自然语言回复
```

### 4.3 工具注册机制

```
@Component + @Tool + @P
        │
        ▼
ToolRegistry.init()  ← @PostConstruct
        │
        ├── 用 getType() 扫描类元数据（不实例化Bean，避免循环依赖）
        ├── 提取 @Tool name / value → 工具名 + 描述
        ├── 从 @P 构建参数 JSON Schema（LLM 不可见的 userId 自动过滤）
        ├── 防冲突：同名工具跳过（warn 日志）
        └── 存入 Map → buildTools() 缓存 → 供 LlmService 使用
```

### 4.4 事件机制（ToolExecutionContext）

```
工具执行时：                               回复发送时：
@Tool 方法                                sendReply()
   │                                         │
   ├── image_gen     → recordImageGen() →    ├── doImageGen()
   ├── image_transform→recordImageTransform→ ├── doImageTransform()
   ├── file_*        → recordFile*()    →    ├── doFileToolAction()
   ├── file_export   → recordFileExport()→   ├── doFileExport()
   ├── re_examine    → recordReExamine()→    ├── doReExamine()
   ├── enable_voice  → VoicePreferenceService.enableVoice()
   └── disable_voice → VoicePreferenceService.disableVoice()

所有事件通过 ThreadLocal 传递，sendReply() 开头自动 getAndClear()。
handleMessage() 入口处也有安全清空，防止异常泄漏。
```

## 六、开发指南

### 6.1 团队协作规则

| 规则 | 说明 |
|------|------|
| `bot/` | **禁止修改**，组长维护 |
| `tool/` | **自由创建**，一人一个文件，零冲突 |
| `service/` | 新增业务服务先同步组长 |
| 工具命名 | 下划线，全局唯一，如 `send_email` |

### 6.2 添加新工具步骤

**只创建一个文件，不改任何其他代码。**

```java
package com.Myself.demo.tool;

import dev.langchain4j.agent.tool.P;
import dev.langchain4j.agent.tool.Tool;
import org.springframework.stereotype.Component;

@Component
public class YourTool {

    @Tool(name = "tool_name", value = "工具描述——LLM靠这个判断是否调用，务必写准")
    public String yourMethod(
            @P("参数描述，带示例值，如：城市名称，如：北京") String param1,
            @P("参数描述") int param2) {

        try {
            // 业务逻辑
            return "操作成功的结果";
        } catch (Exception e) {
            return "操作失败：" + e.getMessage();
        }
    }
}
```

重启后 ToolRegistry 自动扫描注册。

### 6.3 编码规范

| 规则 | 正确 | 错误 |
|------|------|------|
| 返回值 | `String` | `MyObject` |
| 参数类型 | `String`, `int`, `boolean` | `List<Object>` |
| @Tool 描述 | `"查询指定城市的实时天气"` | `"天气"` |
| @P 描述 | `"城市名称，如：北京"` | `"城市"` |
| try-catch | `return "操作失败：xxx"` | 抛 RuntimeException |

### 6.4 CLI 命令扩展

如需 CLI 也支持新命令，让工具类实现 `Command` 接口：

```java
@Component
public class YourTool implements Command {

    @Override public String getName() { return "yourcmd"; }

    @Override public String execute(String[] args) {
        return yourMethod(args[0], Integer.parseInt(args[1]));
    }

    @Tool(name = "your_tool", value = "...")
    public String yourMethod(@P("参数1") String p1, @P("参数2") int p2) { ... }
}
```

## 七、历史问题与解决

| 序号 | 问题 | 解决方案 |
|------|------|---------|
| 1 | 每次启动需扫码 | SessionManager 持久化 LoginContext |
| 2 | 自然语言无法触发命令 | LLM Function Calling |
| 3 | 手动 buildTools() 冗长 | @Tool 注解 + ToolRegistry 自动扫描 |
| 4 | if-else 分发维护难 | ToolRegistry 反射统一执行 |
| 5 | 工具结果直接返回用户 | ReAct 循环：回填 LLM 生成自然语言 |
| 6 | 图片/文件用魔法前缀耦合 | ToolExecutionContext ThreadLocal 事件 |
| 7 | 语音/音色/文件导出/图片重识别用关键词 | 全部改为 @Tool，删除 ~30 处关键词匹配 |
| 8 | CLI 命令与工具类重复定义 | WeatherTool / HelpTool 实现 Command 接口，共用 |
| 9 | 图片文字分开发送 | 图片缓冲机制 |
| 10 | 对话上下文膨胀 | Redis + Token阈值 + 滑动窗口 + 摘要压缩 |
